<!-- Slide 3: What is Context Engineering? -->

# Context Engineering

> **Context Engineering** is the discipline of systematically preparing AI coding assistants for complex tasks by providing comprehensive, structured information‚Äîincluding rules, examples, patterns, and validation mechanisms‚Äîrather than relying on clever prompts alone.

**Key Insight:**
üí° Most AI failures aren't due to model limitations‚Äîthey're due to **insufficient context**.

**The Solution:**
Build a complete information system that teaches the AI how your project works.

# CLAUDE.md - Your Project Constitution

```markdown
# My Project - AI Coding Guidelines

## Stack
- Language: TypeScript (strict mode)
- Framework: Next.js 14 (App Router)
- Database: PostgreSQL + Prisma ORM
- Testing: Jest + React Testing Library

## Code Style
- Use functional components with hooks
- Always use TypeScript types (no any)
- File naming: kebab-case.tsx

## Testing Requirements
- Unit tests for all business logic
- Integration tests for API routes
- Minimum 80% coverage

## Known Gotchas
- Always use 'dynamic' in route handlers
- Prisma queries must use await
- Env vars need NEXT_PUBLIC_ prefix
```

<!-- Slide 13: Context vs Prompts -->

# Why Context > Prompts

**Prompt Engineering:**
```
"Create a REST API endpoint using FastAPI with JWT
authentication, PostgreSQL database integration,
rate limiting at 100 req/min, comprehensive error
handling, and pytest tests with 80% coverage."
```

**Problems:**
- Brittleness (change project ‚Üí prompt breaks)
- Not scalable (can't fit all conventions)
- No memory (starts from scratch each time)
- No validation (can't verify correctness)

**Context Engineering:**
- Persistent rules (CLAUDE.md)
- Reusable patterns (examples/)
- Structured plans (PRPs)
- Built-in validation (at every step)

<!-- Slide 14: Mental Model -->

# Think of It Like Onboarding

**When a Senior Developer Joins Your Team:**

1. **Show them the codebase** (structure, conventions, testing)
2. **Provide examples** ("Here's how we built similar features")
3. **Give detailed requirements** (user stories, acceptance criteria, edge cases)
4. **Enable validation** (linting, tests, code review process)

**Context Engineering does the same for AI assistants.**

It's not about asking better‚Äîit's about providing better information.

# Writing Effective PRPs

**Option 1: Manual** (15-30 min)
- For small features
- Copy `PRPs/templates/prp_base.md`
- Fill in each section

**Option 2: /generate-prp** (2-5 min + review)
- For complex features
- Write `INITIAL.md`

- Review and refine generated PRP

**Hybrid Approach (Recommended):**

# Example: Adding JWT Authentication

**INITIAL.md:**
```markdown
FEATURE: Add JWT authentication to all API endpoints

All /api/ endpoints require valid JWT in Authorization header.
Exclude /api/auth/login and /api/auth/register.

Token includes: user_id, email, role, exp
Expiration: 24 hours

EXAMPLES:
- examples/middleware/auth-check.ts
- examples/api/protected-route.ts

DOCUMENTATION:
- jose library: npmjs.com/package/jose
- Next.js middleware: nextjs.org/docs/...

GOTCHAS:
- Return 401 for missing/invalid tokens
- Verify signature, not just decode
- Log failed auth attempts
```

**Generated PRP includes:**
1. Create JWT utilities (sign/verify)
2. Create auth middleware
3. Update all API tests with auth tokens
4. Add auth logging

<!-- Slide 22: Common Pitfalls -->

# Troubleshooting Common Issues

## Issue: AI ignores CLAUDE.md rules
**Solution:** Be more specific. Use negative examples.
- ‚ùå Bad: "Write good tests"
- ‚úÖ Good: "pytest fixtures, test_ prefix, Given-When-Then"

## Issue: Generated PRPs incomplete
**Solution:** Improve INITIAL.md detail. Include specific examples.

## Issue: Validation loop (AI stuck)
**Solution:** Check test correctness. Add specific guidance in PRP.

## Issue: Wrong file structure
**Solution:** Add explicit tree to CLAUDE.md showing structure.
